核心原则：每个 Spoon 节点拆分为多个独立 Task，每个 Task 只测试一组参数配置。 Spoon 层失败时自动触发 Interface 层（GM指令）复核，精准定位是脚本 Bug还是底层实现 Bug。

StartTaskBTNpcTraceUnit2 — 跟随

NPC 跟随玩家，保持距离区间，障碍物绕行
5 Tasks P0
# Task 名称 Spoon节点配置 代码参数 通过条件
1 Follow_基础跟随 填写「跟随距离=3」「最大距离=8」 followDis=3 maxDis=8 NPC始终保持在 3~8m 范围内跟随移动
2 Follow_到达即停止 勾选「启用结束条件」 enableEndCondition=true NPC到达目标点后停止，不继续游荡
3 Follow_持续跟随不停止 取消勾选「启用结束条件」 enableEndCondition=false NPC持续跟随，无论玩家静止仍保持跟随状态
4 Follow_卡住重置 填写「卡住打断时间=3」 stuckBreakTime=3 卡住3秒后触发重置逻辑，NPC重新找路
5 Follow_超距传送 填写「传送距离=20」 teleportDis=20 距离超过20m时NPC直接传送到玩家附近

StartBTCheckPointMove2 — 引路 / 路点移动

NPC 沿路点移动，可作为引路者在前方带路
5 Tasks P0
# Task 名称 Spoon节点配置 代码参数 通过条件
1 Lead_引路模式 勾选「是否带路」 isLeadingWay=true NPC在玩家前方引路，玩家偏离时等待
2 Lead_普通路点移动 取消勾选「是否带路」 isLeadingWay=false NPC按路点顺序移动，不等待玩家
3 Lead_路点循环 勾选「是否沿路点移动」，勾选「循环」 IsWayPointMove=true loop=true NPC到达终点后循环回起点
4 Lead_引路等待超时 勾选「是否带路」，填写「等待超时=10」 isLeadingWay=true waitTimeout=10 等待玩家超过10s后触发超时回调
5 Lead_指定终点停止 取消勾选「是否沿路点移动」，填写终点坐标 IsWayPointMove=false endPoint NPC移动到指定坐标后停止行为树

StartTaskBTNpcMove2 — 点对点移动

NPC 移动到指定目标点，支持跑 / 走，到达后可播放动作
4 Tasks P1
# Task 名称 Spoon节点配置 代码参数 通过条件
1 Move_跑步到达 「移动方式」选择「跑步」 moveType=Run NPC以跑步速度到达目标点
2 Move_走路到达 「移动方式」选择「行走」 moveType=Walk NPC以行走速度到达目标点
3 Move_到达后播放动作 填写「到达动作=wave」 arriveAction="wave" NPC到达后立即播放指定动作，动作完成后退出
4 Move_到达精度 填写「到达判定半径=0.5」 arriveRadius=0.5 NPC在0.5m误差内触发到达事件

NpcChangeDirection — 转向

NPC 转向目标 / 指定朝向，支持平滑转向和瞬转
4 Tasks P1
# Task 名称 Spoon节点配置 代码参数 通过条件
1 Turn_朝向玩家 「目标类型」选择「玩家」 targetType=Player NPC转向并面朝玩家，误差 < 5°
2 Turn_朝向固定角度 「目标类型」选择「角度」，填写「角度=90」 targetType=Angle angle=90 NPC转到世界坐标90°方向
3 Turn_平滑转向 勾选「平滑转向」，填写「转向速度=90」 smooth=true turnSpeed=90 NPC以 90°/s 平滑旋转，无突变
4 Turn_瞬间转向 取消勾选「平滑转向」 smooth=false NPC在一帧内完成转向

Phase 1 · Q2 2026

全节点接入

6个核心节点全部基础接入
  • StartTaskBTNpcTraceUnit2 基础 Task 接入
  • StartBTCheckPointMove2 基础 Task 接入
  • StartTaskBTNpcMove2 基础 Task 接入
  • NpcChangeDirection 基础 Task 接入
  • SpoonNpcGoOn 基础 Task 接入
  • SpoonNpcGetOff 基础 Task 接入
  • 两层验证框架搭建（Spoon + Interface）
  • 集成进 CI 日报
目标：所有节点跑通，建立基线
Phase 2 · Q3 2026

参数完善 & 关联节点

各节点关键参数覆盖 + 二期关联节点
  • 各节点重要参数拆分为独立 Task
  • Spoon 配置项与代码参数双重标注
  • 跨节点组合场景（引路 + 到达 + 转向）
  • Interface 层复核路径完善
  • 失败率统计与分类归因
二期关联节点（非本侧重点，简单覆盖）
POI 节点（可能影响路点和引路逻辑）、NPC Tag 相关节点（影响 NPC 状态筛选）等，纳入回归验证范围，不作参数级精细拆分。
Phase 3 · Q4 2026

边界测试 & 稳定性

极值参数 + 异常场景 + 长时稳定
  • 极值参数测试（超大距离、极短超时、零值）
  • 卡住/超时异常注入
  • NavMesh 断裂场景复现
  • 长时间运行（30min+）稳定性
  • 多 NPC 并发场景
  • 回归测试套件沉淀
目标：线上回归 0 漏报
